# Claude Code 向け Git 運用ルール

## ブランチ命名規則

```
feature/<issue番号>-<簡単な説明>
fix/<issue番号>-<簡単な説明>
chore/<簡単な説明>
```

### ルール

- `feature/` — 新機能追加
- `fix/` — バグ修正
- `chore/` — コード変更を伴わない作業（依存関係更新、設定変更など）
- `<簡単な説明>` はケバブケース（小文字・ハイフン区切り）で記述する
- GitHub Issue が存在する場合は必ず `<issue番号>` を含める
- Claude Code が自動生成するブランチ（`claude/` プレフィックス）はそのまま使用してよい

### 例

```
feature/12-add-fitbit-sync
fix/7-healthplanet-timestamp
chore/update-uv-dependencies
```

## コミットメッセージ規則

### フォーマット

```
<概要>（1行、50文字以内）

<詳細説明>（必要な場合のみ。変更の背景・理由・注意点を記載）
```

### ルール

- 1行目は **体言止めまたは動詞の命令形** で記述する（例：「〜を追加」「〜を修正」「〜を削除」）
- 1行目は 50文字以内 を目安にする
- 詳細説明が必要な場合は、1行目と空行を挟んで記述する
- コードの「何を」変えたかより「なぜ」変えたかを優先して書く
- Issue に紐づく変更は末尾に `fixes #<番号>` または `refs #<番号>` を付ける

### 変更種別のプレフィックス（任意）

| プレフィックス | 用途                         |
| ------------ | ---------------------------- |
| `feat:`      | 新機能追加                   |
| `fix:`       | バグ修正                     |
| `chore:`     | ビルド・依存関係・設定の変更  |
| `refactor:`  | 動作を変えないコード整理      |
| `docs:`      | ドキュメントのみの変更        |
| `test:`      | テストの追加・修正            |

### 例

```
feat: HealthPlanet の体脂肪率を Fitbit へ同期する機能を追加

計測値が複数ある場合は最新のものを使用する。
refs #24

fix: 23:59 問題を修正（HealthPlanet の計測時刻を Fitbit に連携）

HealthPlanet が返す日付は YYYY-MM-DD 形式のため、
Fitbit に送る際のタイムスタンプが 00:00 になっていた。
計測時刻フィールドを使って正確な時刻を送るよう修正。
fixes #18
```

## PR 作成ルール

- タイトルはコミットメッセージの1行目と同じ形式で記述する
- 本文の冒頭に `Close #<issue番号>` を記載する（Issue が存在する場合）
- 本文には以下を含める
  - 変更の概要（箇条書き）
  - 動作確認の手順またはテスト方法
- マージ前に CI が通っていることを確認する
